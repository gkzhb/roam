Ctrl-C 中断处理在 Aider TUI 界面中的实现

基于测试文件分析，Aider 的 TUI (文本用户界面) 在处理 Ctrl-C 中断时采用了完整的机制来确保用户体验的一致性和应用状态的完整性。

主要处理机制

在 Aider 的 TUI 界面中，Ctrl-C 中断的处理主要通过捕获 KeyboardInterrupt 异常来实现。系统会在不同场景下处理这些中断：

1. 输入提示时的中断处理

当用户在输入提示时按下 Ctrl-C，InputOutput 类会确保多行模式状态被正确恢复：

# 测试中断后多行模式是否恢复  
def test_multiline_mode_restored_after_interrupt(self):  
    io = InputOutput(fancy_input=True)  
    io.prompt_session = MagicMock()  
      
    # 设置为多行模式  
    io.multiline_mode = True  
      
    # 模拟 prompt() 引发 KeyboardInterrupt  
    io.prompt_session.prompt.side_effect = KeyboardInterrupt  
      
    # 测试 confirm_ask()  
    with self.assertRaises(KeyboardInterrupt):  
        io.confirm_ask("Test question?")  
    self.assertTrue(io.multiline_mode)  # 应该被恢复

2. LLM 响应处理中的中断

当在处理 LLM 响应时按下 Ctrl-C，Coder 类会保留部分响应并确保对话历史保持有效状态：

def test_keyboard_interrupt_handling(self):  
    with GitTemporaryDirectory():  
        io = InputOutput(yes=True)  
        coder = Coder.create(self.GPT35, "diff", io=io)  
          
        # 模拟消息处理期间的键盘中断  
        def mock_send(*args, **kwargs):  
            coder.partial_response_content = "Partial response"  
            coder.partial_response_function_call = dict()  
            raise KeyboardInterrupt()  
          
        coder.send = mock_send  
          
        # 初始有效状态  
        sanity_check_messages(coder.cur_messages)  
          
        # 处理将触发中断的消息

        # 验证消息仍处于有效状态  
        sanity_check_messages(coder.cur_messages)  
        self.assertEqual(coder.cur_messages[-1]["role"], "assistant")

3. 确保消息完整性

Aider 特别注重在中断后确保消息结构的完整性，这保证了用户体验的一致性：

主要功能

基于测试文件分析，Aider 的 Ctrl-C 中断处理包括以下关键功能：

  1. 状态保持与恢复 - 在中断后恢复 TUI 的多行模式和其他状态
  2. 部分响应保存 - 在 LLM 生成回复时中断，系统会保留已生成的部分响应
  3. 会话一致性 - 确保中断后对话历史记录保持有效状态
  4. 优雅降级 - 即使在中断期间，系统也能保持消息结构的完整性
